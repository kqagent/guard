Aegis guardrail

Aegis, live guardrail on the real kdb+ estate

Real execution — nothing on this page is pre-recorded

An LLM agent never sends q code to the database. It sends a request (structured) or a hand-written query (free-form); the gate validates every field against the policy allowlist, injects the mandatory bounds and this analyst's row entitlement, and emits the only q that runs — or refuses, fail-closed. Both routes pass through one chokepoint (aegis.query_gate.QueryGate) against a real ~500M-row HDB at ~/aegis-hdb/fsp1. Principal: analyst-equities, entitled to sym in AAPL,MSFT. Each panel shows the same four things: request → Aegis decision → the exact q (or rejection) → real result from the live HDB.
